# 14) العقائد / Doctrines

    - العقائد الثابتة (غير قابلة للتفاوض)
        - **الامتثال أولًا**: لا إطلاق لفئة أو مدينة أو دولة دون ضوابط قابلة للتدقيق (قوانين التجارة، النقل، الصحة، حماية المستهلك، الضرائب).
        - **الأمن والخصوصية بالتصميم**: تقليل البيانات، تشفير، تحكم في الوصول، فصل بيئات، تدقيقات، واحتفاظ بيانات واضح.
        - **الجودة قبل النمو**: لا توسع قبل استقرار مؤشرات: الالتزام بالوقت، الإلغاء، الرضا، النزاعات، ومساهمة ربحية مباشرة.
        - **الشفافية وقابلية التدقيق**: سجل أحداث شامل، أثر قرار، وإثباتات تشغيل (إثبات التسليم) ومسارات نزاعات قابلة للمراجعة.
        - **حيادية السوق ومنع الإقصاء**: قواعد عادلة للشركاء، منع تلاعب بالترتيب، وفصل الإعلان عن البحث قدر الإمكان.
        - **الاقتصاديات أولًا**: القرارات تُبنى على اقتصاديات الوحدة (تكلفة آخر ميل + تكلفة خدمة + خسائر/احتيال) وليس على نمو خام.
        - **قابلية النقل بين الدول**: نواة منصة واحدة + إعدادات محلية (التهيئة المحلية) دون تفريع منتج لكل دولة.
    - عقائد تشغيلية (التميّز التشغيلي)
        - **كل فئة = تشغيل + امتثال + قياس** (حزمة جاهزة: إجراءات، تدريب، هدف مستوى الخدمة وأهداف مستوى الخدمة، ضوابط، لوحات قياس).
        - **تشغيل مدفوع بالبيانات**: قرارات السعة، النوافذ الزمنية، والتسعير تُدار عبر قياس لحظي وتجارب مضبوطة.
        - **إدارة حوادث بمعيار عالمي**: تصنيف شدة، زمن استجابة، تحليل سبب جذري، وإجراءات تصحيحية ووقائية (إجراءات تصحيحية ووقائية).
        - **موثوقية قبل الخصائص**: أي ميزة جديدة تمر ببوابة موثوقية (مراقبة، إنذارات، قابلية رجوع، واختبارات).
        - **هدف مستوى الخدمة واضح للشركاء**: التزام تجهيز، جودة تعبئة، بدائل، ودقة مخزون، مع حوافز/غرامات عادلة.
        - **لوجستيات عكسية معيارية**: إرجاع/استبدال/استدعاء منتجات، مع تتبع وتوثيق.
    - عقائد منتج (المنتج)
        - **تجربة واحدة موحّدة**: بحث → مقارنة → دفع → تتبع → دعم، بنفس المنطق عبر كل الفئات.
        - **تقليل الاحتكاك دون كسر السلامة**: خطوات قليلة، لكن ضوابط إضافية للفئات الحساسة (عمر، وصفات، تبريد).
        - **ثقة المستخدم أصل أساسي**: نزاعات مبنية على أدلة، سياسات واضحة، وإشعارات استباقية.
        - **ولاء عبر قيمة حقيقية**: دقة، بدائل ذكية، شفافية أسعار، ووقت تسليم يمكن الاعتماد عليه.
    - عقائد تقنية (التقنية)
        - **بنية معيارية قابلة للتوسع**: نطاقات واضحة (النطاقات) + واجهات واجهات برمجة التطبيقات ثابتة + فصل مسؤوليات.
        - **سحابة/بنية مرنة**: قابلية توسع تلقائي، عزل متعدد المناطق، وخطط تعافٍ من الكوارث حسب أولوية الخدمة.
        - **منصة أحداث (مدفوع بالأحداث)**: سجل أحداث + رسائل + طوابير لضمان الاعتمادية وفصل المكونات.
        - **إطلاقات آمنة**: مفاتيح تفعيل الخصائص، تجارب ألف/باء، إطلاق تدريجي، وخطة رجوع.
        - **ملاحظة وتشخيص (الملاحظة والتشخيص)**: سجلات + مقاييس + آثار تتبع + مؤشرات أداء رئيسية وربطها بتجربة المستخدم.
        - **بيانات وتحليلات عالمية**: مستودع تحليلي، طبقة تعريف موحدة للمقاييس، وإخفاء هوية عند التحليلات.
        - **ذكاء اصطناعي تطبيقي**: توقع طلب، توصيات، تحسين مسارات، وكشف احتيال، مع حوكمة نماذج (تتبع، انحياز، مراقبة).
