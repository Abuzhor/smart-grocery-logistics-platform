# 19) معجم المصطلحات / Glossary

    - الهدف: توحيد الألفاظ داخل الوثيقة وقاعدة البيانات، بحيث يكون لكل مفهوم اسم عربي ثابت يُستخدم في جميع الأقسام.
        
        
        | المصطلح العربي المعياري | المعنى المعياري داخل المشروع | أين يُستخدم |
        | --- | --- | --- |
        | نواة المنصة | قدرات أساسية مشتركة تخدم كل الفئات ولا تتغير بتغير السوق | التقنية، التشغيل، الإطلاق |
        | إجراءات تشغيل معيارية | خطوات تشغيل مفصلة قابلة للتكرار لكل فئة ومدينة | التشغيل، الشراكات، الجودة |
        | محرك السياسات | قواعد تشغيل قابلة للتهيئة حسب الدولة والمدينة والفئة | الامتثال، إطلاق الدول، المخاطر |
        | اتفاقية مستوى الخدمة | التزامات زمن وجودة بين المنصة والشركاء والقوى العاملة | التشغيل، الشراكات، المؤشرات |
        | أهداف مستوى الخدمة | أهداف داخلية تُدار بها جودة التشغيل يوميًا | التشغيل، المراقبة |
        | الرصد التشغيلي | مراقبة الأداء والأعطال والتنبيه المبكر والتحقيق | التشغيل، الحوادث |
        | الثقة والسلامة | ضوابط الاحتيال والنزاعات والسلامة وحماية المستخدم | المدفوعات، النزاعات، الامتثال |
        | إثبات التسليم | توثيق استلام وتسليم الخدمة أو البضاعة بزمن وموقع ودليل | القوى العاملة، النزاعات، الامتثال |
        | الحجز المؤقت | احتجاز مبلغ مؤقتًا حتى اكتمال التسليم أو حل النزاع | المدفوعات، بين الأفراد، النزاعات |
        | اقتصاديات الوحدة | ربحية كل طلب بعد تكلفة الخدمة وليس بعد المصاريف العامة | الإيرادات، قرارات التوسع |
        | بوابات القرار | معايير عبور إلزامية قبل الانتقال لمرحلة توسع جديدة | الإطلاق العالمي |
        | حزمة القرار | قالب قرار يربط المتطلبات بالمؤشرات والمخاطر وخطة الإطلاق والرجوع | آلية التحسين |
        | واجهة المستخدم وتجربة المستخدم | تصميم ما يراه المستخدم وكيف يتفاعل مع الخدمة لضمان سهولة الاستخدام وتقليل الأخطاء | التصميم، الاختبارات، التحسين المستمر |
        | نماذج أولية تفاعلية | تصميمات قابلة للنقر لمحاكاة تجربة المنتج قبل التطوير | التصميم، البحث، التحقق من المتطلبات |
        | مفاتيح تفعيل الخصائص | آلية لتفعيل أو إيقاف خاصية دون تحديث التطبيق، لتجارب آمنة وإطلاق تدريجي | الإطلاق، التجارب، إدارة المخاطر |
        | بنية وحدات | تصميم يجزّئ النظام إلى وحدات مستقلة لتسهيل الصيانة والتوسع | التقنية، قابلية التوسع |
        | مستودع بيانات تحليلي | طبقة مخصصة للتحليلات والتقارير تجمع البيانات وتتيح الاستعلامات بسرعة | القياس، التقارير، قرارات التوسع |
        | ذاكرة تخزين مؤقت | تخزين سريع للبيانات الأكثر طلبًا لتقليل زمن الاستجابة وخفض التكلفة | الأداء، الاستقرار |
        | محرك بحث تشغيلي | بحث سريع ومتقدم داخل الكتالوج والطلبات والمخزون | تجربة المستخدم، الكتالوج |
        | ناقل رسائل للأحداث | نظام يبث الأحداث بين المكونات لرفع الاعتمادية وفصل الأجزاء | التقنية، التشغيل، الرصد |
        | طوابير رسائل | تنظيم معالجة المهام على دفعات لضمان عدم فقدان الرسائل والتحكم في الحمل | التقنية، الإشعارات، تكاملات المدفوعات |
        | تحسين المسارات | حساب أفضل مسار لتقليل الوقت والتكلفة وزيادة عدد المهام المنجزة | الإسناد، القوى العاملة |
        | تقدير وقت الوصول | توقع وقت وصول الطلب بناءً على البيانات والازدحام والسعة | تجربة المستخدم، التشغيل |
        | تحديد نطاقات تشغيل | تقسيم المدينة إلى نطاقات لتوزيع الطلبات والسعة والسياسات بوضوح | الإطلاق، التشغيل، الامتثال |
        | إدارة النوافذ الزمنية | تخصيص أوقات تسليم وحجز سعة لضمان الالتزام بالوقت | التشغيل، التجربة، الأعمال إلى أعمال |
        | جدولة التحميل والتنزيل | حجز فترات زمنية للاستلام والتفريغ في مراكز الإتمام والمستودعات | سلاسل الإمداد، الأعمال إلى أعمال |
        | التحقق من الهوية | إثبات هوية المستخدم أو الشريك وفق ضوابط الدولة لتقليل الاحتيال ورفع الثقة | التسجيل، المدفوعات، الامتثال |
        | التحقق من الشريك التجاري | تحقق من نظامية التاجر أو المزود (سجل تجاري، ترخيص، عنوان، حساب مصرفي) | الشراكات، الامتثال، المخاطر |
        | مكافحة الاحتيال | مجموعة ضوابط وتقنيات لاكتشاف ومنع السلوكيات الاحتيالية قبل وقوع الضرر | المدفوعات، الثقة والسلامة، التشغيل |
        | إدارة النزاعات | إجراءات لحل خلافات التسليم والجودة والمدفوعات وفق قواعد واضحة وأدلة | الدعم، الثقة والسلامة، الامتثال |
        | سياسة الإلغاء | قواعد إلغاء الطلب بحسب المرحلة والوقت والفئة مع رسوم أو استرجاع وفق الضوابط | التجربة، التشغيل، الاقتصاديات |
        | سياسة الاسترجاع | قواعد إعادة المبالغ كليًا أو جزئيًا وفق سبب الاسترجاع والأدلة | المدفوعات، النزاعات، الاقتصاديات |
        | اللوجستيات العكسية | إرجاع البضائع أو العبوات أو الاستبدال وإعادة إدخالها لسلسلة الإمداد | سلاسل الإمداد، التشغيل، الجودة |
        | تسوية المدفوعات | تحويل مستحقات الشركاء والقوى العاملة وفق دورة زمنية بعد خصم الرسوم والمطالبات | المدفوعات، الشراكات، الاقتصاديات |
        | تقسيم الطلب | تجزئة طلب واحد إلى أجزاء بحسب المزود أو الفئة مع إبقاء تجربة موحدة للمستخدم | تنسيق الطلبات، التجربة، التشغيل |
        | تجميع المهام | ضم عدة مهام متقاربة زمنًا ومكانًا في جولة واحدة لرفع الكفاءة | الإسناد، القوى العاملة، الاقتصاديات |
        | تخطيط السعة | تقدير وتوزيع السعة التشغيلية المطلوبة حسب الطلب المتوقع والموارد المتاحة | الإطلاق، التشغيل، الجودة |
        | مؤشر رضا العميل | قياس ولاء المستخدم عبر سؤال احتمالية التوصية وتحويله إلى مؤشر رقمي | القياس، التجربة، التحسين |
        | مساهمة ربحية مباشرة | الربح بعد طرح تكلفة تقديم الخدمة مباشرة من الإيراد قبل المصاريف العامة | الاقتصاديات، بوابات القرار |
        | حوكمة البيانات | قواعد ملكية البيانات وصلاحيات الوصول والاحتفاظ والتدقيق وحماية الخصوصية | الامتثال، التقنية، الثقة والسلامة |
        | تصنيف البيانات | تقسيم البيانات إلى درجات حساسية لتطبيق ضوابط مناسبة لكل درجة | الأمن، الامتثال، حوكمة البيانات |
        | سجل التدقيق | تتبع آلي لمن فعل ماذا ومتى على البيانات والعمليات لسهولة المراجعة | الامتثال، الحوادث، الثقة والسلامة |
        | إخفاء الهوية | إزالة أو تشفير عناصر تعريف الشخص عند استخدام البيانات للتحليلات | الجهات، التحليلات، الامتثال |
        | إدارة الحوادث | منهجية للتعامل مع الأعطال والأحداث المؤثرة من الاكتشاف حتى الإغلاق مع توثيق الدروس | التشغيل، الرصد التشغيلي، السجلات |
        | شدة الحادث | تصنيف يحدد خطورة الحادث وأولوية الاستجابة وزمن المعالجة | الحوادث، التشغيل، بوابات القرار |
        | تحليل السبب الجذري | تحديد السبب الحقيقي للحادث أو المشكلة لمنع تكرارها بدل معالجة الأعراض | الحوادث، الجودة، التحسين |
        | إجراء تصحيحي | تغيير يُنفذ لإزالة سبب المشكلة بعد وقوعها وتقليل احتمال تكرارها | الجودة، الحوادث، التشغيل |
        | إجراء وقائي | تغيير يُنفذ لمنع مشكلة محتملة قبل وقوعها بناءً على مؤشرات إنذار مبكر | المخاطر، الجودة، التشغيل |
        | إدارة الشكاوى | استقبال شكاوى المستخدمين وتحليلها وربطها بجودة التشغيل وإغلاقها ضمن زمن محدد | الدعم، الجودة، الرصد التشغيلي |
        | اتفاقية شراكة | اتفاق يحدد مسؤوليات الشريك والتزاماته التشغيلية والمالية ومتطلبات التكامل | الشراكات، الامتثال، الاقتصاديات |
        | صحة الشريك | مؤشر مركب يقيس التزام الشريك بالجودة والوقت ونسبة الأخطاء والشكاوى | الشراكات، القياس، بوابات القرار |
        | تدقيق الجودة | فحص دوري للتحقق من التزام الشركاء والإجراءات بمعايير الجودة المعتمدة | الجودة، الشراكات، الامتثال |
        | معايير الجودة | مجموعة ضوابط ومقاييس تحدد الحد الأدنى المقبول لجودة الخدمة لكل فئة | الجودة، التشغيل، الإطلاق |
        | التحكم في الوصول | تحديد من يحق له رؤية البيانات أو تعديلها بناءً على الدور والصلاحيات | حوكمة البيانات، الامتثال، الأمن |
        | تشفير البيانات | حماية البيانات بتحويلها إلى صيغة غير مقروءة إلا بمفتاح مصرح به | الأمن، الامتثال، الثقة والسلامة |
        | احتفاظ البيانات | تحديد مدة حفظ البيانات وآلية أرشفتها أو حذفها وفق الأنظمة واحتياج العمل | حوكمة البيانات، الامتثال، السجلات |
        | موافقة المستخدم | إذن صريح من المستخدم لمعالجة بياناته أو مشاركتها وفق نطاق محدد | الخصوصية، الامتثال، التجربة |
        | حق الحذف | حق المستخدم في طلب حذف بياناته وفق الأنظمة مع استثناءات الالتزام النظامي | الخصوصية، الامتثال، حوكمة البيانات |
        | سجل التغييرات | توثيق التعديلات على السياسات والإجراءات والأنظمة لضمان التتبع والرجوع | السجل التاريخي، سجل التغيير، الامتثال |
        | خطة رجوع | خطة لإلغاء تغيير أو إصدار وإعادة الوضع السابق دون تعطيل الخدمة | الإطلاق، حزمة القرار، إدارة المخاطر |
        | إطلاق تدريجي | إطلاق التغيير على شريحة صغيرة ثم التوسع تدريجيًا بعد التحقق من المؤشرات | الإطلاق، التجارب، الرصد التشغيلي |
